# Slurm 集群监控系统 - 详细介绍

## 📋 系统概述

Slurm 集群监控系统是一个专为 Slurm 工作负载管理器设计的综合 Web 监控平台。它提供实时的集群状态监控、作业管理、资源使用统计和历史数据分析功能，帮助管理员和用户更好地了解集群运行状况。

---

## 🎯 核心功能

### 1. 仪表盘 (Dashboard)

仪表盘提供集群整体运行状况的一览视图，包括：

#### 节点状态卡片
- **总节点数**: 集群中所有节点的数量
- **空闲节点**: 可接受新作业的节点
- **已分配节点**: 正在运行作业的节点
- **故障节点**: 不可用或离线的节点
- **混合状态节点**: 部分资源被占用的节点

#### 作业状态卡片
- **总作业数**: 当前队列中的所有作业
- **运行中作业**: 正在执行的作业
- **等待中作业**: 队列中等待资源的作业
- **暂停作业**: 被暂停的作业

#### 资源使用率
- **CPU 资源**: 显示总核心数、已分配核心、空闲核心及使用率进度条
- **GPU 资源**: 显示总 GPU 数、已分配 GPU、可用 GPU 及利用率进度条

#### 可视化图表
- **节点状态分布**: 饼图展示各类状态节点的比例
- **作业状态分布**: 饼图展示运行中、等待中、暂停作业的比例
- **CPU 资源使用**: 饼图展示已分配与空闲 CPU 的占比

---

### 2. 节点监控 (Nodes)

#### 节点详细信息表
| 字段 | 说明 |
|------|------|
| 节点名 | Slurm 节点标识 |
| 状态 | idle(空闲)/alloc(已分配)/mix(混合)/down(故障) |
| 分区 | 节点所属的分区 |
| CPU配置 | 分配的CPU/空闲CPU/其他/总计 |
| 内存 | 总内存容量 |
| 空闲内存 | 当前可用内存 |
| 负载 | 系统负载 |
| GPU | GPU 资源配置 |
| 操作 | 查看详情、排空/恢复节点 |

#### 节点管理功能
- **查看详情**: 点击节点名查看详细配置信息
- **排空节点(Drain)**: 将节点标记为维护状态，不再接受新作业
- **恢复节点(Resume)**: 将节点恢复为可用状态
- **状态筛选**: 按节点状态快速筛选

---

### 3. 作业管理 (Jobs)

#### 标签页切换
作业管理页面提供两个标签页：
- **活跃作业**: 当前运行中、等待中、暂停的作业
- **历史任务**: 过去 30 天内已完成的所有作业

#### 活跃作业表格
| 字段 | 说明 |
|------|------|
| 作业ID | Slurm 作业唯一标识 |
| 名称 | 用户指定的作业名 |
| 用户 | 提交作业的用户 |
| 分区 | 作业运行的分区 |
| 状态 | R(运行中)/PD(等待中)/S(暂停)/CG(完成中) |
| 节点 | 分配的节点数 |
| 节点列表 | 具体分配的节点名 |
| CPU | 请求的 CPU 核心数 |
| 内存 | 请求的内存 |
| GPU | 请求的 GPU 资源 |
| 优先级 | 作业调度优先级 (从 %Q 字段获取) |
| 时间 | 运行时间或等待时间 |
| 操作 | 查看详情、取消作业 |

#### 历史任务表格
展示过去 30 天的已完成作业：

| 字段 | 说明 |
|------|------|
| 作业ID | Slurm 作业唯一标识 |
| 名称 | 用户指定的作业名 |
| 用户 | 提交作业的用户 |
| 账户 | 作业所属账户 (Account) |
| 分区 | 作业运行的分区 |
| 状态 | COMPLETED/FAILED/CANCELLED/TIMEOUT |
| 退出码 | 作业退出代码 (如 0:0 表示成功) |
| 节点 | 分配的节点数 |
| CPU | 请求的 CPU 核心数 |
| GPU | 请求的 GPU 数量 |
| 预留内存 | 作业预留的内存 (ReqMem) |
| 最大内存 | 实际使用的最大内存 (MaxRSS) |
| 磁盘写入 | 最大磁盘写入量 (MaxDiskWrite) |
| 磁盘读取 | 最大磁盘读取量 (MaxDiskRead) |
| 运行时间 | 实际运行时长 (Elapsed) |
| 提交时间 | 作业提交时间 |

**历史任务功能特点：**
- **分页展示**: 默认 50 条/页，支持 50/100 条切换
- **状态筛选**: 已完成 / 失败 / 已取消 / 超时
- **分区筛选**: 选择特定分区查看
- **用户搜索**: 输入用户名快速搜索
- **详情查看**: 点击作业ID查看完整信息

#### 作业详情抽屉
点击任意作业ID打开详情抽屉，参考 `jobinfo` 脚本展示详细信息：

**基本信息**
- 作业ID、状态、用户、账户、分区、QOS
- 优先级、作业名

**资源分配**
- 节点数、CPU 核心、预留内存
- GPU 资源（自动解析 GPU 数量）
- 分配节点列表

**时间信息**
- 提交时间、开始时间、结束时间
- 等待时间（Waited）
- 预留运行时间（Reserved walltime）
- 实际运行时间（Used walltime）

**CPU 使用详情** (已完成作业)
- Used CPU time: CPU 总使用时间
- % User (Computation): 用户态时间占比
- % System (I/O): 系统态时间占比
- CPU 利用率

**内存使用详情** (已完成作业)
- Mem reserved: 预留内存
- Max Mem used: 最大内存使用
- 内存利用率

**磁盘 I/O** (已完成作业)
- Max Disk Write: 最大磁盘写入量
- Max Disk Read: 最大磁盘读取量

**完成状态** (已完成作业)
- ExitCode: 退出码（0:0 表示成功）

**等待原因** (待运行作业)
- Reason: 调度等待原因

**工作目录**
- 完整工作路径

**提交命令** (如 Slurm 版本支持)
- SubmitLine: 完整的提交命令行

**技术实现：**
- 使用 `sacct` 获取历史/已完成作业的详细信息
- 使用 `sstat` 获取运行中作业的实时统计
- 使用 `squeue` 获取待运行作业的等待原因
- 智能显示：仅当有数据时才显示对应信息区域

**数据来源：**
- 使用 `sacct` 命令获取历史数据
- 合并主作业和 `.batch` 步骤的资源使用数据
- 按提交时间倒序排列

#### 作业管理功能
- **批量选择**: 支持多选作业进行批量操作
- **批量取消**: 一键取消多个选中的作业
- **密码验证**: 取消作业时需要管理员密码验证
- **作业筛选**: 按状态、分区、关键词搜索
- **暂停/恢复**: 暂停运行中作业或恢复已暂停作业

| PID | 进程 ID |
| 类型 | C(计算)/G(图形) |

| 显存使用率 | GPU 显存使用百分比 |
| 编码 | 视频编码使用率 |
| 解码 | 视频解码使用率 |
| 命令 | 执行的程序名 |

### 7. 统计报表 (Statistics)

统计报表是系统的核心分析功能，提供多维度的集群使用分析。

#### 时间范围选择
- 过去 24 小时
- 过去 7 天（默认）
- 过去 30 天

#### 7.1 集群使用率概览
四个关键指标卡片：
- ✅ **已完成作业**: 成功完成的作业数量
- ❌ **失败作业**: 执行失败的作业数量
- ⏱️ **总 CPU 时间**: 所有作业消耗的 CPU 时间（小时）
- 📋 **总作业数**: 提交的作业总数

#### 7.2 用户资源使用情况 ⭐

这是系统的核心功能之一，展示每个用户的资源消耗详情：

**表头说明：**
| 列名 | 说明 |
|------|------|
| 用户 | 用户名 |
| 作业数 | 该用户提交的作业总数 |
| CPU所用核时 | CPU 核心使用时间（分钟） |
| 占比 | 该用户 CPU 使用占全集群的百分比 |
| 调用GPU的作业数 | 使用 GPU 资源的作业数量 |
| GPU所用核时 | GPU 核心使用时间（分钟） |
| 占比 | 该用户 GPU 使用占全集群的百分比 |

**显示效果示例：**

```
总计: CPU核时 15234.5 分钟 | GPU核时 3456.2 分钟

| 用户    | 作业数 | CPU所用核时 | 占比    | 调用GPU的作业数 | GPU所用核时 | 占比   |
|---------|--------|-------------|---------|-----------------|-------------|--------|
| user1   | 25     | 5234.5      | ████34.4% | 5               | 1234.5      | ████35.7% |
| user2   | 15     | 3123.2      | ████20.5% | -               | -           | -      |
| user3   | 30     | 2876.8      | ████18.9% | 3               | 987.3       | ████28.6% |
| user4   | 8      | 1567.4      | ████10.3% | -               | -           | -      |
| user5   | 12     | 1234.6      | ███░8.1%  | 2               | 678.9       | ████19.6% |
```

**数据说明：**
- **CPU所用核时**: 计算公式为 `CPU核心数 × 运行时间(分钟)`
  - 例如：使用 4 核运行 30 分钟 = 120 核时
- **GPU所用核时**: 计算公式为 `GPU卡数 × 运行时间(分钟)`
  - 例如：使用 2 张 GPU 运行 60 分钟 = 120 核时
- **占比**: 该用户使用量 ÷ 所有用户总使用量 × 100%
- **进度条**: 蓝色表示 CPU 占比，红色表示 GPU 占比

## 🚀 安装部署

### 依赖要求
- Python 3.7+
- Flask
- Flask-SocketIO
- Slurm 工作负载管理器
- NVIDIA GPU (可选，用于 GPU 监控)

### 快速启动
```bash
# 安装依赖
pip install flask flask-socketio

# 启动服务
python app.py

# 访问
http://localhost:5000
```

### 系统服务部署
使用提供的 systemd 服务文件：
```bash
sudo cp slurm-monitor.service /etc/systemd/system/
sudo systemctl enable slurm-monitor
sudo systemctl start slurm-monitor
```
