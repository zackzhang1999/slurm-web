# 小学刷题网站开发计划

## 项目概述
- **项目名称**: 小学刷题网站
- **目标用户**: 小学三年级至六年级学生
- **科目**: 语文、数学、英语（江苏地区教材）
- **核心功能**: 题目练习、错题训练、竞技PK、用户管理

---

## 一、需求分析

### 1.1 功能需求

| 序号 | 功能模块 | 详细描述 |
|------|----------|----------|
| 1 | 题库管理 | 导入/管理语文/数学/英语题目，每学期每科至少500题 |
| 2 | 用户系统 | 普通用户注册登录、管理员后台 |
| 3 | 练习模式 | 选择科目、选择题目数量进行答题 |
| 4 | 错题训练 | 错题详细解析 + 5道类似题目强化训练 |
| 5 | PK竞技 | 用户正确率排名、排行榜展示 |
| 6 | 分科训练 | 语文/数学/英语分开训练 |

### 1.2 技术栈建议

| 层级 | 技术选型 |
|------|----------|
| 前端 | React + TypeScript + Tailwind CSS |
| 后端 | Node.js + Express |
| 数据库 | MySQL / PostgreSQL |
| 题目来源 | 互联网收集 + 手动录入 |

```typescript
interface Question {
  id: string;
  subject: 'chinese' | 'math' | 'english';
  grade: 3 | 4 | 5 | 6;
  semester: 1 | 2;
  type: 'choice' | 'fill' | 'judge' | 'essay';

  options?: string[];  // 选择题选项
  answer: string;
  explanation: string; // 错题解析
  difficulty: 1 | 2 | 3; // 简单/中等/困难

}
```

## 四、API接口设计

### 4.1 用户接口
| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /api/auth/register | 用户注册 |
| POST | /api/auth/login | 用户登录 |
| GET | /api/users/me | 获取当前用户信息 |

### 4.2 题目接口
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/questions | 获取题目列表 |
| POST | /api/questions | 添加题目（管理员） |
| PUT | /api/questions/:id | 修改题目（管理员） |
| DELETE | /api/questions/:id | 删除题目（管理员） |
| POST | /api/questions/batch | 批量导入题目 |

### 4.3 练习接口
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/practice/start | 开始练习（获取题目） |
| POST | /api/practice/submit | 提交答案 |
| GET | /api/practice/result | 获取练习结果 |

### 4.4 错题接口
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/wrong | 获取错题列表 |
| GET | /api/wrong/:id/similar | 获取5道相似题 |
| POST | /api/wrong/:id/review | 标记已复习 |

### 4.5 排行榜接口
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/ranking | 获取排行榜 |
| GET | /api/ranking/subject/:subject | 分科目排行 |

## 五、开发优先级

1. **高优先级**: 用户系统、题库管理、基础答题功能
2. **中优先级**: 错题训练系统、PK排行榜
3. **低优先级**: 社交功能、成就系统

## 七、注意事项

1. 题目收集需要确保版权合规，仅用于学习目的
2. 用户数据需做好隐私保护
3. 系统需支持高并发访问（学生使用高峰）
4. 移动端体验需优化
5. 需要定期更新题库
