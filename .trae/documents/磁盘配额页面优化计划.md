# 磁盘配额页面优化计划

## 目标
优化磁盘配额页面，新增磁盘空间使用和剩余情况的显示，并根据配额使用情况对用户进行分类显示。

## 功能需求

### 1. 磁盘空间使用和剩余情况显示
- 显示每个用户的磁盘使用量
- 显示磁盘剩余空间
- 用进度条或图表展示使用比例
- 显示软限制和硬限制

### 2. 用户分类显示
根据配额使用情况将用户分为三类：

#### 接近软限用户
- 定义：使用量 >= 软限制的 80% 且 < 软限制
- 显示：黄色警告状态
- 提示：接近软限制，请注意

#### 超限用户（软限和硬限之间）
- 定义：软限制 <= 使用量 < 硬限制
- 显示：橙色警告状态
- 提示：已超出软限制，请尽快清理

#### 通用用户（超过硬限制）
- 定义：使用量 >= 硬限制
- 显示：红色危险状态
- 提示：已超出硬限制，无法写入

#### 正常用户
- 定义：使用量 < 软限制的 80%
- 显示：绿色正常状态

## 技术方案

### 后端 API
- 复用现有的 `/api/disk-quota` 接口
- 返回数据包含：使用量、软限制、硬限制、剩余空间

### 前端页面
- 修改 `templates/disk_quota.html`
- 添加分类筛选功能
- 添加可视化展示（进度条/图表）

### 原型图设计
- 顶部统计卡片：显示总用户数、接近软限、超限、超过硬限的数量
- 筛选按钮：全部、正常、接近软限、超限、超过硬限
- 表格展示：用户、使用量、软限制、硬限制、剩余、使用率、状态
