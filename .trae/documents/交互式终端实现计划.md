# Xterm 终端交互式输入实现计划

## 目标

实现真正的交互式 SSH 终端，允许用户直接在 Xterm.js 终端中输入字符，后端通过 SSH 会话实时处理输入并返回输出。

## 当前问题

* 只能通过下方输入框输入命令

* 每次执行需要等待 HTTP 请求完成

* 无法实现真正的交互式终端体验

## 技术方案

### 架构设计

使用 WebSocket + paramiko 实现交互式 SSH 会话：

1. 前端：Xterm.js 直接捕获键盘输入，通过 WebSocket 发送
2. 后端：维护 SSH 会话，通过 WebSocket 实时传输数据

### 实现步骤

#### 1. 安装 paramiko 库

在 requirements.txt 中添加：

```
paramiko>=3.0.0
```

#### 2. 修改 app.py

* 添加 WebSocket 终端处理函数 (`socketio.on`)

* 创建 SSH 会话管理类

* 实现交互式命令执行

#### 3. 修改终端模板

* 移除下方输入框

* 将 Xterm.js 键盘事件绑定到 WebSocket 发送

#### 4. 安全性

* 仅管理员可使用终端

* SSH 连接超时处理

* 会话清理机制

## 文件修改清单

1. `requirements.txt` - 添加 paramiko
2. `app.py` - 添加 SSH 会话管理和 WebSocket 端点
3. `templates/terminal.html` - 移除输入框，添加 WebSocket 连接

## 风险控制

* SSH 会话超时自动断开

* 异常处理确保资源释放

* 限制并发会话数量
