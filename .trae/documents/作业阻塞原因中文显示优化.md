# 作业管理阻塞原因中文显示优化计划

## 目标
将作业管理页面中的阻塞原因（Reason）从英文转换为中文显示，鼠标悬停时显示详细具体原因。

## 当前问题
- 阻塞原因直接显示英文原始值，如 "Resources"、"Priority" 等
- 用户无法快速理解作业被阻塞的具体原因

## 实施方案

### 步骤 1: 创建原因映射函数
在 `templates/index.html` 中添加函数 `translateJobReason(reason)`，将英文原因映射为中文：

映射规则：
- `Resources` → 资源不足
- `Priority` → 优先级低
- `Dependency` → 等待依赖作业
- `InvalidQOS` → 无效的 QOS
- `QOSGrpLimit` → QOS 组资源限制
- `QOSJobLimit` → QOS 作业数限制
- `QOSTimeLimit` → QOS 时间限制
- `AssocGrpLimit` → 账户组资源限制
- `AssocJobLimit` → 账户作业数限制
- `AssocTimeLimit` → 账户时间限制
- `GrpJobs` → 组作业数已达上限
- `GrpTRES` → 组资源已达上限（CPU/内存/GPU）
- `GrpWall` → 组时间已达上限
- `MaxJobsPerUser` → 用户作业数已达上限
- `MaxTRESPerUser` → 用户资源已达上限（CPU/内存/GPU）
- `MaxWallPerUser` → 用户时间已达上限
- `NodeDown` → 节点故障
- `BadConstraints` → 约束条件不满足
- `PartitionTimeLimit` → 分区时间限制
- `PartitionNodeLimit` → 分区节点不足
- `JobArrayJobLimit` → 作业数组超限
- `SystemFailure` → 系统故障

### 步骤 2: 修改渲染逻辑
在作业表格渲染时，调用映射函数：
- 原值保留在 title 属性中（鼠标悬停显示详细信息）
- 显示区域使用中文翻译

### 步骤 3: 测试验证
- 确认阻塞原因正确显示为中文
- 确认鼠标悬停显示原始英文原因

## 文件修改
