# 统计报表功能增强说明

## 概述
本次更新大幅增强了统计报表页面的功能，添加了多个实用的统计分析功能，包括可视化图表、趋势分析和数据导出等。

## 新增功能

### 1. 概览统计卡片
- **已完成作业数**: 显示指定时间段内成功完成的作业数量
- **失败作业数**: 显示失败的作业数量
- **成功率**: 计算并显示作业成功率百分比
- **CPU时间**: 统计总CPU使用时间（小时）

### 2. 可视化图表

#### 作业状态分布饼图
- 使用饼图展示各种作业状态的分布
- 包括：完成、失败、取消、超时等状态
- 直观展示集群整体健康状况

#### 作业时长分布柱状图
- 按时间段分组统计作业数量
- 分组：0-1小时、1-6小时、6-12小时、12-24小时、24小时以上
- 帮助了解作业运行时长分布

#### 作业提交趋势（24小时）
- 折线图展示最近24小时每小时的作业提交数量
- 帮助识别使用高峰期
- 支持平滑曲线显示

#### 作业提交趋势（按天）
- 柱状图展示最近N天的作业提交数量
- 便于进行日对比分析
- 支持长达90天的数据统计

#### 等待时间分布
- 环形图展示作业等待时间的分布
- 分组：<5分钟、5-30分钟、30分钟-2小时、2-12小时、12小时以上
- 帮助评估调度器效率

### 3. 作业等待时间分析
- **平均等待时间**: 所有作业的平均等待时长
- **中位数等待时间**: 等待时间的中位数（更公平的指标）
- **最短等待**: 最快被调度的作业等待时间
- **最长等待**: 等待时间最长的作业
- **等待分布图**: 可视化展示等待时间的分布情况

### 4. 分区使用情况表
显示每个分区的详细统计：
- 作业总数
- 已完成作业数
- 失败作业数
- 成功率（带进度条可视化）
- CPU使用时间

### 5. 节点效率排行
按资源利用率对节点进行排名：
- 综合效率评分
- 分配率
- 空闲率
- 故障率
- 可视化进度条显示

### 6. 用户排行
两种维度的用户排行：
- **CPU时间排行**: 按CPU使用时间排名
- **作业数排行**: 按提交作业数量排名
- 前三名使用金色/银色/铜色标识

### 7. 数据导出功能
支持导出统计数据为CSV格式：
- **导出作业数据**: 导出指定时间段内的作业记录
- **导出用户数据**: 导出用户统计数据
- 支持浏览器直接下载

## 新增 API 端点

| 端点 | 说明 | 参数 |
|------|------|------|
| `GET /api/stats/job-distribution` | 作业状态分布 | `hours` |
| `GET /api/stats/partition-usage` | 分区使用情况 | `hours` |
| `GET /api/stats/hourly-trend` | 小时趋势 | `hours` |
| `GET /api/stats/daily-trend` | 日趋势 | `days` |
| `GET /api/stats/duration-distribution` | 作业时长分布 | `hours` |
| `GET /api/stats/wait-time-analysis` | 等待时间分析 | `hours` |
| `GET /api/stats/node-efficiency` | 节点效率排行 | `days` |
| `GET /api/stats/resource-efficiency` | 资源利用率 | `hours` |
| `GET /api/stats/account-usage` | 账户使用统计 | `days` |
| `GET /api/stats/export` | 导出CSV | `type`, `hours` |

## 时间范围选择
支持多种时间范围：
- 过去24小时
- 过去7天（默认）
- 过去30天
- 过去90天

所有统计数据都会根据选择的时间范围动态更新。

## 技术实现

### 后端
- 使用 `sacct`、`sreport`、`sinfo` 等 Slurm 命令获取数据
- 并行加载多个统计数据以提高性能
- 使用 `timeout` 防止命令阻塞
- 优雅处理 accounting 未启用的情况

### 前端
- 使用 Chart.js 进行数据可视化
- 动态创建和销毁图表以避免内存泄漏
- 响应式设计，支持各种屏幕尺寸
- 加载动画和过渡效果提升用户体验

## 使用方法

1. 访问统计报表页面：点击侧边栏"统计报表"或访问 `/?page=statistics`
2. 选择时间范围：使用下拉框选择要查看的时间范围
3. 查看图表：所有图表会自动更新
4. 导出数据：点击"导出作业数据"按钮下载CSV文件
5. 刷新数据：点击刷新按钮或开启自动刷新

## 注意事项

1. **Accounting 数据**: 部分统计功能需要 Slurm Accounting 数据库支持。如果未启用，会回退到使用 squeue 数据。
2. **性能考虑**: 大时间范围（如90天）的统计可能需要较长时间加载。
3. **权限**: 部分统计信息可能需要管理员权限才能查看完整数据。
